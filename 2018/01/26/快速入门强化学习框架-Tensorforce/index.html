快速入门强化学习框架-Tensorforce

Tensorforce 强化学习
 2018/01/26   Share

在想要使用深度强化学习的应用中，TensorForce 可以作为一个库使用。它提供了一种声明式接口，用户通过配置文件，设定要使用的强化学习算法和参数，以及网络结构，就能快速实验不同的强化学习算法，而无需担心所有底层的设计。更具体的介绍可以查看这篇文章<<基于TensorFlow打造强化学习API：TensorForce是怎样炼成的？>> 这里介绍下使用Tensorforce的经验，方便大家可以快速地用深度学习来训练自己的应用。

安装

在Tensorforce的github地址，按照ReadMe文件指导，安装Tensorforce框架。最方便方式是用pip安装，如果之前没安装tensorflow，先安装tensorflow，再安装tensorforce。

pip install tensorflow

pip install tensorforce

安装完成后，可以在github地址上，下载example，运行下。

python examples/openai_gym.py CartPole-v0 -a examples/configs/ppo.json -n examples/configs/mlp2_network.json

创建环境

example运行成功后，就可以着手写自己的应用了。首先需要写一个环境文件。这里我们以海盗和宝藏的为例子。共有8个state，用0-7标志。走到5，7的海盗时，reward减1。走到6的宝藏，reward加1。共有4个动作，从左->上->右->下分别设为0，1，2，3。

在创建环境时，可以参考tensorforce/contrib/openai_gym.py文件。主要修改以下几个函数：

init(self）：初始化函数。这个根据自己的应用，初始化一些内部变量。

class MazeEnv(Environment):    
    def __init__(self):
        """
        Initialize maze.

        """
        self.seed = np.random.RandomState()
        self.nowstate = self.seed.randint(0,4)

        self.terminal_state = dict()
        self.terminal_state[5] = 1
        self.terminal_state[6] = 1
        self.terminal_state[7] = 1

        self.rewards = dict()
        #key: nowstate_action
        self.rewards['0_3'] = -1.0
        self.rewards['2_3'] = 1.0
        self.rewards['4_3'] = -1.0

        self.trans = dict()
        self.trans['0_3'] = 5
        self.trans['0_2'] = 1
        self.trans['1_0'] = 0
        self.trans['1_2'] = 2
        self.trans['2_0'] = 1
        self.trans['2_2'] = 3
        self.trans['2_3'] = 6
        self.trans['3_0'] = 2
        self.trans['3_2'] = 4
        self.trans['4_0'] = 3
        self.trans['4_3'] = 7

close(self)：结束函数。应用结束后，做销毁结构体操作。我们应用比较简单，这里就直接pass掉。

reset(self)：重置函数。在每轮episode结束后，重置环境，函数需要返回重置后的observation。一开始我没返回observation，卡了很久，最后只能去看源代码。

def reset(self):
   self.nowstate = self.seed.randint(0,5)
   return self.nowstate

execute(self,actions)： 执行函数。执行actions操作后，环境需要做的改变，函数返回下一个state、episode是否结束、reward。

def execute(self, actions):      
    if self.nowstate in self.terminal_state:
        return self.nowstate, True, 0
    key = '%d_%d' % (self.nowstate, actions)
    if key in self.trans:
        next_state = self.trans[key]
    else:
        next_state = self.nowstate
    terminal = False
    if next_state in self.terminal_state:
        terminal = True
    self.nowstate = next_state
    if key not in self.rewards:
        r = 0.
    else:
        r = self.rewards[key]
    return next_state, terminal, r

states(self)：状态表示的形式。这里我们把state直接用离散值表示，如果状态是多维的连续空间，可以参考注释里的形式。

@property
def states(self):    
    return dict(shape=(), type='int')
    #return dict(shape=tuple((2,)),type='float')

actions(self)：动作的表示形式，这里也是把动作表示成离散值。

@property
    def actions(self):
       return dict(type='int', num_actions=4)

主函数

至此环境配置完毕，然后编写主函数，可以参考github地址下的example/openai_gym.py。主要是修改起始的参数配置和应用环境以及episode_finished函数。episode_finished是每一轮episode结束后会调用的函数，可以打印环境信息，追踪模型的效果。

#解析参数
parser = argparse.ArgumentParser()
parser.add_argument('-a', '--agent-config', help="Agent configuration file")
parser.add_argument('-n', '--network-spec', default=None, help="Network specification file")
parser.add_argument('-e', '--episodes', type=int, default=100000, help="Number of episodes")
parser.add_argument('-t', '--timesteps', type=int, default=30000000, help="Number of timesteps")
parser.add_argument('-m', '--max-episode-timesteps', type=int, default=1000, help="Maximum number of timesteps per episode")
parser.add_argument('-d', '--deterministic', action='store_true', default=False, help="Choose actions deterministically")
parser.add_argument('-D', '--debug', action='store_true', default=False, help="Show debug outputs")

args = parser.parse_args()
handler = logging.handlers.RotatingFileHandler("./log/maze.log", maxBytes=1024 * 1024, backupCount=5)
logger = logging.getLogger(__name__)
logger.setLevel(logging.INFO)
logger.addHandler(handler)
logger.info("At the beginning of the Main")

environment = maze_env.MazeEnv()

运行时，命令行配置为
-a ./conf/dqn.json -n ./conf/mlp2_embedding_network.json

这里dqn.json和mlp2_embedding_network.json，参考的是examples/configs下的配置。另外还有一点需要注意，早期的tensowforce暂不支持state为离散状态的输入，所以有人在网络层那加了一层embedding。但是这个修改还没加到发行版，所以如果是用pip install tensowforce安装的框架的话，需要去github上，找到tensorforce/core/networks目录下的init.py和layer.py两个文件，替换本地tensowforce安装目录下对应的文件。

到这里整个应用就实现好了，剩下的就是在运行的时候，再不断去调整配置文件下的参数。一开始我跑了1000轮episode，发现算法并不收敛，平均reward只是上下波动。初步怀疑是次数不够，所以又试了10000轮，但是仍没有进展。想了想那可能是网络那边设置出了问题，所以又停下代码，去学习了下李飞飞的深度学习。在学了几天后，终于放弃了课程，硬着头皮去调试代码。在黄博大神的指导下，轮了100W次，终于看到reward有上升的趋势。后面发现是参数那边设置得不是很match。默认配置里面，探索概率是从1依次递减到0.1，而递减要经过的轮数是100W轮…也就说前面的1W轮，基本都是处于随机探索过程。
